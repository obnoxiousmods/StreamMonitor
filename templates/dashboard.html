REST API Explorer
Health & Status
GET /api/ping Unauthenticated liveness check
Returns a simple pong response to verify the server is running. No authentication required.
Example response:
{"pong": true}
GET /api/public Unauthenticated health summary
Public health summary with service counts and uptime. No authentication required.
Example response:
{"total": 12, "up": 11, "down": 1, "uptime_pct": 91.7}
GET /api/status Auth Full service health + history
Returns health status, latency, and history for all monitored services.
Example response:
{"services": {"comet": {"alive": true, "latency_ms": 42, ...}}, "timestamp": "..."}
GET /api/status/{service_id} Auth Single service health
Returns health status and history for a single service by its ID.
Example response:
{"alive": true, "latency_ms": 42, "history": [1,1,1,0,1,...]}
GET /api/stats Auth All service statistics
Returns API statistics (requests, latency, errors) for all monitored services.
Example response:
{"comet": {"requests": 1523, "avg_ms": 38, ...}, ...}
GET /api/stats/{service_id} Auth Single service stats
Returns API statistics for a single service by its ID.
Example response:
{"requests": 1523, "avg_ms": 38, "errors": 2}
GET /api/versions Auth Installed vs latest versions
Returns installed and latest GitHub release versions for each service.
Example response:
{"comet": {"installed": "1.2.3", "latest": "1.2.4"}, ...}
Logs & Errors
GET /api/logs/{unit} Auth Journal log lines
Returns recent journal log lines for the specified systemd unit. Supports ?n=200 to control line count.
Example response:
{"lines": ["Mar 28 12:00:01 host comet[1234]: ...", ...]}
GET /api/dmesg Auth Kernel logs
Returns kernel log output (journalctl -k). Useful for diagnosing hardware and driver issues.
Example response:
{"lines": ["Mar 28 12:00:00 kernel: ...", ...]}
GET /api/errors Auth Scanned error history
Returns all scanned errors and warnings from monitored services.
Example response:
{"errors": [{"service": "comet", "severity": "error", "line": "...", "ts": "..."}], "count": 5}
POST /api/errors/scan Auth Trigger error scan
Triggers an immediate scan of all service logs for errors and warnings.
Example response:
{"ok": true, "scanned": 12, "found": 3}
DELETE /api/errors Auth Clear error history
Clears all stored error and warning history.
Example response:
{"ok": true, "cleared": 15}
Service Control
POST /api/service/{unit}/{action} Auth Start/stop/restart a service
Controls a systemd service. Actions: start, stop, restart. Unit is the systemd unit name.
Example response:
{"ok": true, "unit": "comet", "action": "restart"}
Settings
GET /api/settings/keys Auth List API keys
Returns all configured API keys (values are masked for security).
Example response:
{"keys": {"TMDB": "abc...xyz", "Real-Debrid": "***...***"}}
POST /api/settings/keys Auth Update API keys
Updates one or more API keys. Send a JSON body with key-value pairs.
Example request body:
{"TMDB": "new_api_key_here"}
GET /api/settings/urls Auth List service URLs
Returns all configured service URLs used for health checks.
Example response:
{"urls": {"comet": "http://localhost:8191", ...}}
POST /api/settings/urls Auth Update service URLs
Updates one or more service URLs. Send a JSON body with service-URL pairs.
Example request body:
{"comet": "http://localhost:8191"}
POST /api/settings/password Auth Change password
Changes the admin password. Requires current password for verification.
Example request body:
{"current": "old_pass", "new": "new_pass"}
Tools
GET /api/benchmark?imdb=ttXXXXXXX Auth Run addon benchmark
Runs a benchmark against configured addons for the given IMDB title. Returns latency, stream counts, and resolution breakdown.
Example response:
{"title": "...", "results": [{"name": "comet", "latency_ms": 1200, "streams": 15, ...}]}
GET /api/jellyfin Auth Sessions & activity
Returns active Jellyfin sessions and recent activity log.
Example response:
{"sessions": [...], "activity": [...]}
POST /api/perms/scan Auth Scan permissions
Scans configured directories and reports ownership/permission mismatches.
Example response:
{"sections": [{"name": "Media", "items": [{"path": "/data", "ok": true, ...}]}]}
POST /api/perms/fix Auth Fix permissions
Applies ownership and permission fixes to selected paths. Send a JSON body with paths and desired permissions.
Example request body:
{"paths": ["/data/media"], "owner": "joey:joey", "mode": "775"}
GET /speedtest Auth Speed test page
Serves the speed test HTML page for running download/upload tests.
Example: Open in browser
HTML page
GET /speedtest/download?mb=25 Download endpoint (rate-limited)
Returns a binary payload of the specified size for speed testing. No authentication required but rate-limited. Adjust mb parameter for payload size.
Example response:
Binary data (25 MB)
